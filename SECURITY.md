# Security

## Reporting a vulnerability

If you believe you have found a security vulnerability in FruityWolf, please report it responsibly.

**Do not** open a public GitHub issue for security-sensitive bugs.

**Preferred options:**

1. **GitHub Security Advisories** — Use [GitHub’s security advisory feature](https://github.com/FruityWolf/FruityWolf/security/advisories/new) for this repository to report the issue privately.
2. **Private contact** — If the repository lists maintainer contact information (e.g. in README or profile), you may contact maintainers privately with a clear description of the issue and steps to reproduce.

Please include:

- A short description of the vulnerability
- Steps to reproduce
- Impact (what an attacker could do)
- Any suggested fix, if you have one

We will acknowledge your report and work with you to understand and address it. We ask that you do not disclose the issue publicly until it has been addressed (e.g. a fix or advisory has been released).

## Data and scope

FruityWolf runs locally and does not send your data to external servers. Configuration and library data are stored in your app data directory (e.g. `%APPDATA%\FruityWolf` on Windows). If you are concerned about sensitive paths or data in bug reports, please redact them or report security issues through the private channels above.

Thank you for helping keep FruityWolf and its users safe.
